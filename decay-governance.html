Decay is not deletion. It is lifecycle management. Context that was once essential to a decision should never disappear — but it should not continue driving live reasoning after the conditions it described have resolved.

Decay Governance

Context Relevance Across Time

Manages the lifecycle of contextual facts. The problem is vertical — context that was true becoming less relevant as the world it described changes.

ProblemRelevance degradation over time
DirectionVertical — across time
UrgencyManaged — predictable and gradual
GovernsContext Graph lifecycle
OutputState: Active → Archived → Deprecated
Without itStale context contaminates live reasoning

Both problems are silent. Neither announces itself. Drift accumulates through legitimate local decisions. Decay accumulates as the world changes around static context records. Without active governance, both are invisible until a decision goes wrong.

The Three Decay States

State 01 — Active

Currently Relevant

Influences live reasoning. Visible to agents and analysts when synthesizing recommendations or evaluating performance. Context remains Active until an explicit trigger moves it — a resolution event, a time threshold, or a conditions-changed flag.

Duration: weeks → years depending on context type
State 02 — Archived

No Longer Current

Excluded from live reasoning. Fully preserved for lineage reconstruction and historical analysis. Carries its full provenance: what it was, when it was active, what triggered the transition, and what replaced it. Queryable — just not present in current-state views.

Retention: indefinite, for decision lineage integrity
State 03 — Deprecated

Superseded

Replaced by new context. The transition point is recorded. The deprecated record points forward to what replaced it; the replacement points back. This is a labeled handoff — not erasure. The bidirectional pointer preserves the narrative of how understanding evolved.

Pointer: bidirectional, timestamped, preserved
Why Deletion Is the Wrong Answer Stale context was the episodic record at the moment of a past decision. Deleting it makes that decision permanently uninterpretable. The recommendation made in August 2024 cannot be reconstructed without the competitive context that was active — and then resolved — in that period. Deletion destroys lineage. Archival preserves it.
Real Estate Application
RE Application

When Drift and Decay Interact

The most complex governance scenario is when both problems occur simultaneously. A definition of "market exposure" was used consistently for three years, then drifted — different teams started applying it differently. Simultaneously, the market conditions it was designed to capture have changed.

You now have both problems at once: the term is semantically inconsistent and the context it described is no longer current. Resolving the drift by picking one canonical definition is not enough — you also need to archive the contextual snapshots that each version was applied to, so that historical analysis remains coherent across the version boundary.

The Governance Rule When a drift event is resolved with a formal version boundary, that boundary should trigger a decay state review of any context tagged with the prior definition version. The two governance disciplines must coordinate — drift resolution and decay archival are not independent events.
