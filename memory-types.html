The three types
of memory

Three distinct types, each contributing something the others cannot. Memory does not reduce to a single capability. Semantic, episodic, and procedural memory operate independently, and all three are required for an architecture that genuinely remembers.

SM

Semantic Memory

Meaning · Stability · Relationship

Knowledge of what things mean, concepts, categories, relationships, and the rules connecting them. It is general, not situational. It holds the framework that makes events interpretable, not the events themselves.

It is timeless in character. Definitions, ontologies, taxonomies, and governed metrics belong here. Semantic memory is what allows a term defined in one period to be interpreted correctly in another.

Lives In Ontology and Semantic Layer. Requires version governance, when a definition changes, the prior version must be preserved. Monitored by the Drift Register for divergence across counterparties and systems.
EM

Episodic Memory

Circumstance · Time · Context

The record of what happened, when, under what conditions, with what result. It is autobiographical and time-stamped. It answers "what did we do?" not "what does it mean?"

A log from 2019 describes what happened, but if the definitions used in 2019 have since drifted, the log becomes uninterpretable. The record survives. The meaning doesn't. Episodic memory without semantic grounding decays into uninterpretable noise.

Lives In Context Graph, but only if explicitly temporal. Requires the context graph to record what was active at the moment a decision was made, not just at present. Governed by Decay Logic: context states transition through Active → Archived → Deprecated.
PM

Procedural Memory

Execution · Judgment · Audit

How things are done, workflows, rules, sequences of action. It carries execution, not explanation. A workflow runs correctly even if no one can articulate why those steps exist. This is its power and its danger.

Elevated to the reasoning layer, procedural memory becomes the auditable record of judgment. How the system synthesized meaning, measurement, and context into a recommendation at a specific moment in time.

Lives In Reasoning layer. Requires binding of output to the state of all lower layers at the moment of execution. Without this binding, conclusions are orphaned from the premises that produced them, and cannot be audited, challenged, or built upon.
Semantic Memory Answers
What does it mean, and which version of that definition applies?
Episodic Memory Answers
What happened, when, under what conditions, and is that context still current?
Procedural Memory Answers
How was it done, by what logic, and what was the state of the system when judgment was produced?
Real Estate Application
RE Application

Each Memory Type in a Real Estate Portfolio

Semantic memory in real estate: knowing that "net rentable area" and "gross leasable area" are distinct concepts, that "cap rate" means different things in stabilized versus value-add contexts, and that "tenant" in a commercial lease occupies a structurally different position in the domain model than "customer" in retail analytics. It also means knowing that JLL and Greystar may use the same word "occupancy" to mean different things, and that both definitions are valid within their respective contexts.

Episodic memory in real estate: the property management transition at 400 Market Street in Q1 2024. The deferred HVAC decision logged in June 2024. The competitor running 3-month free rent in the Austin submarket from August through November 2024. The rising rate macro context that shaped hold/sell recommendations from 2022 through mid-2025. These facts shaped decisions at specific moments, and they decay in relevance as conditions change.

Procedural memory in real estate: the underwriting model that applies a specific discount rate methodology to value-add industrial assets in secondary markets. The model ran correctly for three years. But when the methodology changed in Q1 2024, historical underwriting outputs became incomparable to current ones, because no version binding was in place to record which methodology was active when each deal was modeled.

The Consequence Without all three memory types operating together, a cycle comparison becomes unreliable. You can retrieve the deals underwritten in 2022. You cannot reliably compare their underwriting assumptions to deals underwritten in 2025, because the definitions, the contextual conditions, and the modeling methodology have all changed without a consistent record of when and how.
